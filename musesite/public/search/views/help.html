Search Documentation

The search interface allows you to mine our corpus for projects using complex search queries. We have provided a set of pre-constructed saved searches to help get you started or you can manually construct your search query using the provided interactive query builder located in the left sidebar navigation panel. The search queries consist of 3 distinct types of filters; project filters, commit filters, and file filters.

Download Results

After doing a search, use the Download JSON button to download search results.This will download a JSON file containing the relative paths of the projects that match your search query. These project paths can be used to copy projects off of the XtreemFS mount.

Search Syntax

  • Case Sensitivity
    • By default all searches are case insensitive. To perform case sensitive searches surround your search value with double quotes. Wildcards are supported.

         ie) "HeapSort" or "someFunctionName"

  • Wildcards
    • All string search fields support wildcards. The supported wildcards are * and ? --where * matches any character sequence (including the empty one), and ?, matches any single character. Note this query can be slow, as it needs to iterate over many terms. In order to prevent extremely slow wildcard queries, a wildcard term should not start with one of the wildcards * or ?. 

         ie) android* or off?by?one

  • Phrase Matching
    • The ability to search for a phrase of words is supported however this is only supported for a few search fields. See tables below. To perform phrase matching searches surround your word phrase in double qoutes. Wildcards not supported in phrase matching. 

         ie) "off by one"

    Project Filters

    Allows you to filter on top level project criteria.

    # Field Type Supported syntax
    1 name string case sensitive, wildcards
    1 description string wildcards, phrase match
    2 language string multi-select
    3 total_size int >, <, =, >=, <=
    4 isAndroidApp boolean true, false
    5 readme string wildcards, phrase match
    6 install string wildcards, phrase match
    7 site string multi-select

    Commit Filters

    Allows you to search all commit logs from a given project. (only supported for github projects)

    # Field Type Supported syntax
    1 id long >, <, =, >=, <=
    1 message string wildcards, phrase match
    2 parent_id long >, <, =, >=, <=
    3 child_id long >, <, =, >=, <=

    File Filters

    Allows you to search all source files within a project by selected criteria.

    # Field Type Supported syntax
    1 comment string wildcards, phrase matching
    1 file_extension string case sensitive, wildcards
    2 file_name string case sensitive, wildcards
    3 function.name string case sensitive, wildcards
    4 function.return_type string case sensitive, wildcards
    5 function.parameter_type string case sensitive, wildcards
    6 variable.name string case sensitive, wildcards
    7 variable.type string case sensitive, wildcards
